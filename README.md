# 개발 워크스테이션 구축 실습

## 1, 개요

이 과제는 개발 환경을 직접 구성하여 터미널, Docker, Git/GitHub의 기본 기능을 실습하고 이해하는 것을 목표로 하고 있습니다.</br>
간단한 명령어와 권한 설정, Docker 컨테이너 운영 및 Dockerfile 빌드와 포트 매핑 볼륨 관리 등을 수행하고 재현 가능한 환경을 구성하는 실습입니다.

## 2. 실행 환경

- OS: macOS 26.3.1
- Shell: zsh
- Docker: 28.5.2
- Git: 2.51.0

## 4. 디렉토리 구조

codyssey </br>
├── image/ # 실습 증거 스크린샷 </br>
├── web # Dockerfile 빌드 및 Nginx 정적 페이지 테스트 폴더</br>
| ├── Dockerfile </br>
| └── index.html </br>
├── bind # 바인드 마운트 실습용 폴더</br>
| └── index.html </br>
└── </br>

- **image/**  
  실습 증거를 명확히 남기기 위해 단계별 스크린샷을 기능(1~10단계)별로 정리했습니다.  
  문서 재현성과 평가 편의를 위해 별도 폴더로 분리했습니다.

- **web/**  
  Dockerfile 빌드 실습을 위해 Nginx 기반 정적 웹 서버를 만드는 공간입니다.  
  Dockerfile과 COPY될 HTML 파일을 한 폴더에 모아 빌드 경로 충돌을 방지했습니다.

- **bind/**  
  바인드 마운트 실습을 독립적으로 수행하기 위해 web과 분리된 폴더에 구성했습니다.  
  로컬HTML 수정 → 컨테이너 내부 반영 과정을 명확하게 보기 위함입니다.

   #### chmod 숫자 구조 설명 (예: 755, 644)

   chmod는 세 자리 숫자로 구성되며</br />
   각 숫자는 “사용자/그룹/기타”의 권한을 의미합니다.</br />

   예: 755 = rwx r-x r-x </br />

   - 7 = 4(r) + 2(w) + 1(x) → 사용자 권한 </br />
   - 5 = 4(r) + 1(x) → 그룹 </br />
   - 5 = 4(r) + 1(x) → 기타</br />

   ##### docker run -d -p 8081:80 -v $(pwd):/경로 --name bind-container nginx - 바인드 마운트로 현재 폴더 컨테이너와 연결

   간단 설명 : </br>

   ##### 로컬 파일을 수정하면 컨테이너를 재시작하지 않아도 즉시 변경 내용이 반영되는 것을 확인함.

   #### 절대 경로 vs 상대 경로 선택 기준

   - 절대 경로(/Users/...): 특정 PC 환경에 종속됨
   - 상대 경로($(pwd)): 현재 디렉토리를 자동으로 가져오므로  
     **어떤 PC에서도 동일하게 재현 가능**

   ##### 바인드 마운트 실습에서 $(pwd)를 사용한 이유는 실습을 재현하는 사람이 경로를 일일이 수정하지 않아도 되도록 하기 위함.

## 6. 트러블 슈팅(2건)

#### 1. 포트 충돌 발생

바운드 마운트 실습을 할 때 포트가 충돌된 상황이 발생하였습니다. 원인으로는 Dockerfile을 실습할 때 사용했던 8080 포트를 삭제하지 않은 상태에서 사용할려고 하여 포트가 충돌하여 오류가 나는 상태가 나왔습니다. 해결 방안으로 가장 먼저 8080을 삭제하고 다시 시도하여 성공하였고 다른 방법으로는 8080 포트를 대신하여 8081 포트를 사용하는 방법 2가지를 이용하였습니다.

#### 2. git 충돌 발생

연동을 하고 난 후 git push 명령어를 입력했을 때 오류가 발생 하였습니다. 오류의 원인으로는 github에서 수정한 내용을 로컬 환경에서 pull하지 않은 상태에서 push를 한 상태였습니다. 해결 방안으로는 먼저 로컬 환경에서 pull을 사용하여 변경한 내용을 받아오고 그 후에 push를 하는 방법을 이용하였습니다.
